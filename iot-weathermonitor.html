Project Overview

This project reads temperature and humidity from a DHT11 sensor connected to an ESP8266 (NodeMCU). The board connects to Wi‚ÄëFi and sends readings to the Blynk cloud so they can be displayed live on a smartphone dashboard.

  • Sensor: DHT11 (temperature & humidity)
  • Microcontroller: NodeMCU / ESP8266
  • App: Blynk IoT (mobile app + cloud)
  • Purpose: Learn about IoT sensors, Wi‚ÄëFi, and data monitoring

Wiring (simple)

  • DHT11 VCC ‚Üí 3V3 on NodeMCU
  • DHT11 GND ‚Üí GND on NodeMCU
  • DHT11 DATA ‚Üí D4 (GPIO2) on NodeMCU

How the Code Works (Summary)

The ESP8266 connects to Wi‚ÄëFi and sends temperature and humidity readings from the DHT11 sensor to Blynk every few seconds.

  • Libraries used: ESP8266WiFi.h, BlynkSimpleEsp8266.h, and DHT.h.
  • Auth token links the hardware to your Blynk project. (Example: "YourAuthTokenHere")
  • Wi‚ÄëFi setup: Replace ssid and pass with your own Wi‚ÄëFi credentials in code (e.g., ssid[] = "YourWiFi";).
  • DHT sensor: Reads humidity and temperature using dht.readHumidity() and dht.readTemperature().
  • Blynk virtual pins: Temperature ‚Üí V5, Humidity ‚Üí V6 for displaying on app widgets.

Blynk Dashboard Setup

  1. Install the Blynk IoT app.
  2. Create a new Template and Device.
  3. Add two Value Display widgets: bind V5 for temperature and V6 for humidity.
  4. Power your NodeMCU, open the app, and observe live readings.

Troubleshooting & Tips

  • Double-check wiring and ensure the DHT11 module is powered by 3.3V.
  • Replace the Auth Token with your own from Blynk when cloning the project.
  • Ensure stable Wi‚ÄëFi connection and good USB cable for powering the NodeMCU.
  • Keep the DHT11 away from direct heat for accurate readings.

Quick Setup Checklist

  1. Wire DHT11 to NodeMCU (VCC ‚Üí 3V3, GND ‚Üí GND, DATA ‚Üí D4)
  2. Install Arduino IDE & ESP8266 board support
  3. Install libraries: Blynk, DHT
  4. Insert Wi‚ÄëFi credentials and Blynk token
  5. Upload code and open Serial Monitor (9600 baud)
